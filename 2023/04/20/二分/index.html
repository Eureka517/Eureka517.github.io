整数二分

思路(区间套寻找分界点)

  1. 找到一个性质使得能够分为满足和不满足的两段,使用mid来缩小分界点的范围，直到范围内只有一个点，即分界点

  2. 选取左段的右边界/右段的左边界作为分界点(因为是整数二分，所以左段的右边界=右段的左边界-1)

    1. 选取左段的右边界
      1. mid=l+r+1 >>1
      2. 若mid在左段中，则分界点范围[l,r]缩小为[mid,r]，即l=mid；
      3. 若mid在右段中，则分界点范围[l,r]缩小为[l,mid-1]；

  3. 选取左段的右边界/右段的左边界作为分界点

    1. 选取右段的左边界
      1. mid=l+r>>1
      2. 若mid在左段中，则分界点范围[l,r]缩小为[mid+1,r]；
      3. 若mid在右段中，则分界点范围[l,r]缩小为[l,mid]，即r=mid；
  • 注：关于l+r+1>>2什么时候需要+1，考虑极限情况l=r-1，此时第一种情况会使得mid=l，从而死循环。可以先写代码，如果是l=mid，则需要+1

模板

  1. 左段的右边界
    while (l < r) //右段的左边界模板
    {
      int mid = l + r >> 1;//写完再根据情况判断要不要+1
  if (q[mid] >= x)r = mid;//二分的结果为满足>=x的最小值
  else l = mid + 1;
    }
  2. 右段的左边界
    while (l < r) //左段的右边界模板
    {
      int mid = l + r + 1 >> 1;//由于此时是l = mid，则需要+1
  if (q[mid] <= x)l = mid;
  else r = mid - 1;
    }

浮点数二分

思路

二分缩小区间，因为时小数，最后l和r往往不能完全相等，判定条件为r-l<1e-x(x为题目要求精度+2)，即两者差距小于一定值是认定二者相等

  • 注：相较于整数二分，不需要考虑取整(+1)的问题

模板

double l = -10000, r = 10000;//理解为在-10000~10000的范围内找到需要的数
while(r-l>=1e-8)
{ 
    double mid = (l + r)/2;//此时无需考虑+1的问题
    if (mid * mid * mid >= x)r = mid;
    else l = mid;
}
